# project_spending_block コンバーター仕様書

## 1. 概要

このドキュメントは、`project_spending_block` テーブルにデータを挿入するためのCSVファイルを生成するPythonスクリプト `convert_project_spending_block.py` の仕様を記述します。

## 2. 目的

`source_data/source_test_data/5-1_mockdata.csv` ファイルから関連データを抽出し、`project_spending_block` テーブルのスキーマに準拠した形式で `converted_data/converted_project_spending_block.csv` ファイルを生成します。

## 3. 入力ファイル

- **ファイルパス**: `source_data/source_test_data/5-1_mockdata.csv`
- **エンコーディング**: UTF-8
- **主要なカラム**:
    - `事業年度`: `budget_year` に対応
    - `予算事業ID`: `project_id` に対応
    - `支出先ブロック番号`: `block_no` に対応
    - `支出先ブロック名`: `block_name` に対応
    - `事業を行う上での役割`: `role_in_project` に対応
    - `ブロックの合計支出額`: `block_total_amount` に対応

## 4. 出力ファイル

- **ファイルパス**: `converted_data/converted_project_spending_block.csv`
- **エンコーディング**: UTF-8
- **カラム**:
    - `project_id` (SMALLINT)
    - `budget_year` (SMALLINT)
    - `block_no` (VARCHAR(8))
    - `block_name` (VARCHAR(255))
    - `role_in_project` (VARCHAR(1000))
    - `block_total_amount` (NUMERIC(14,2))

## 5. 変換ロジック

1.  **入力ファイルの読み込み**: `source_data/source_test_data/5-1_mockdata.csv` をCSV辞書リーダーで読み込みます。
2.  **データ抽出と変換**:
    *   各行から `事業年度`, `予算事業ID`, `支出先ブロック番号`, `支出先ブロック名`, `事業を行う上での役割`, `ブロックの合計支出額` の値を取得します。
    *   `事業年度`, `予算事業ID`, `支出先ブロック番号` のいずれかが欠けている行はスキップします。
    *   `事業年度` と `予算事業ID` は整数型に変換します。変換に失敗した場合は、その行をスキップします。
    *   `ブロックの合計支出額` は、カンマを除去し、浮動小数点数に変換します。値がない場合や変換に失敗した場合は `None` とします。
3.  **重複排除**:
    *   `(project_id, budget_year, block_no)` の組み合わせを一意のキーとして使用し、重複するブロックデータは出力しません。最初に見つかった組み合わせのみが採用されます。
4.  **出力ファイルの書き込み**: 変換されたデータを `converted_data/converted_project_spending_block.csv` に書き込みます。ヘッダー行も含まれます。

## 6. データベーススキーマとの関連

生成されるCSVファイルは、以下の `project_spending_block` テーブルのDDLに準拠しています。

```sql
CREATE TABLE IF NOT EXISTS project_spending_block (
  block_id           BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  project_id         SMALLINT   NOT NULL,
  budget_year        SMALLINT   NOT NULL,          -- 今回は 2024 を投入
  block_no           VARCHAR(8) NOT NULL,          -- 'A','B',...
  block_name         VARCHAR(255),
  role_in_project    VARCHAR(1000),
  block_total_amount NUMERIC(14,2),
  created_at         TIMESTAMP DEFAULT now(),

  CONSTRAINT fk_psb_project
    FOREIGN KEY (project_id, budget_year)
    REFERENCES project(project_id, budget_year)
      ON UPDATE RESTRICT ON DELETE RESTRICT
);

CREATE UNIQUE INDEX IF NOT EXISTS uq_psb_project_block
  ON project_spending_block(project_id, budget_year, block_no);
```

**`block_id` について**: `block_id` は `BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` として定義されており、データベースが自動的に値を生成するため、CSVファイルには含めません。
