# PostgreSQL データインポート手順書

この手順書は、生成されたCSVファイルをPostgreSQLデータベースにインポートする詳細な手順を説明します。特に、`project_spending_block` および `project_spending` テーブルへのデータインポートに焦点を当てます。

## 1. 前提条件

*   PostgreSQLサーバーが稼働しており、アクセス可能な状態であること。
*   データベースユーザーが、対象テーブルへの `INSERT` 権限を持っていること。
*   `psql` コマンドラインツールが利用可能であること。
*   `DB_data/DDL.sql` に基づいてテーブルが既に作成されていること。
*   `converted_data/converted_project_spending_block.csv` および `converted_data/converted_project_spending.csv` ファイルが生成済みであること。

## 2. インポートの全体フロー

1.  **`project_spending_block` テーブルへのインポート**:
    *   `converted_data/converted_project_spending_block.csv` を `project_spending_block` テーブルにインポートします。この際、`block_id` はデータベースによって自動生成されます。
2.  **`project_spending` テーブルへのインポート**:
    *   `converted_data/converted_project_spending.csv` を直接インポートするのではなく、`project_id`, `budget_year`, `block_no` をキーとして `project_spending_block` テーブルから対応する `block_id` を取得し、その `block_id` を含めて `project_spending` テーブルにデータを挿入するスクリプトを実行します。

## 3. `project_spending_block` テーブルへのデータインポート

`project_spending_block` テーブルは `block_id` が `GENERATED BY DEFAULT AS IDENTITY` であるため、CSVファイルには `block_id` を含めずにインポートします。

### 手順

1.  **PostgreSQLに接続**:
    ```bash
    psql -h your_host -p your_port -U your_user -d your_database
    ```
    （例: `psql -h localhost -p 5432 -U postgres -d gyousei_db`）

2.  **CSVファイルをインポート**:
    `COPY` コマンドを使用してCSVファイルをインポートします。`block_id` 列は指定しません。

    ```sql
    \COPY project_spending_block (project_id, budget_year, block_no, block_name, role_in_project, block_total_amount) FROM 'converted_data/converted_project_spending_block.csv' WITH (FORMAT CSV, HEADER TRUE, ENCODING 'UTF8');
    ```
    *   `project_id, budget_year, block_no, block_name, role_in_project, block_total_amount`: CSVファイルの列の順序と一致させる必要があります。
    *   `converted_data/converted_project_spending_block.csv`: 生成されたCSVファイルのパス。
    *   `HEADER TRUE`: CSVファイルの1行目がヘッダーであることを示します。
    *   `ENCODING 'UTF8'`: ファイルのエンコーディングを指定します。

3.  **インポートの確認**:
    データが正しくインポートされたかを確認します。

    ```sql
    SELECT COUNT(*) FROM project_spending_block;
    SELECT * FROM project_spending_block LIMIT 10;
    ```

## 4. `project_spending` テーブルへのデータインポート

`project_spending` テーブルは `block_id` を外部キーとして参照しており、この `block_id` は `project_spending_block` テーブルにデータがインポートされた後に生成されます。そのため、直接CSVをインポートするのではなく、Pythonスクリプトを使用して `block_id` をルックアップしながらデータを挿入します。

### 手順

1.  **Pythonスクリプトの準備**:
    以下の内容で `import_project_spending.py` というPythonスクリプトを作成します。このスクリプトは `psycopg2` ライブラリを使用するため、事前にインストールが必要です (`pip install psycopg2-binary`)。

    ```python
    # import_project_spending.py
    import csv
    import os
    import psycopg2
    from psycopg2 import extras

    # データベース接続情報
    DB_HOST = "localhost"
    DB_NAME = "gyousei_db" # あなたのデータベース名に置き換えてください
    DB_USER = "postgres"   # あなたのユーザー名に置き換えてください
    DB_PASSWORD = "your_password" # あなたのパスワードに置き換えてください
    DB_PORT = "5432"

    def import_project_spending_data(source_csv_path):
        conn = None
        try:
            conn = psycopg2.connect(
                host=DB_HOST,
                database=DB_NAME,
                user=DB_USER,
                password=DB_PASSWORD,
                port=DB_PORT
            )
            cur = conn.cursor()
            
            print(f"Attempting to import project_spending data from: {source_csv_path}")

            with open(source_csv_path, mode='r', encoding='utf-8') as infile:
                reader = csv.DictReader(infile)
                
                if not reader.fieldnames:
                    print("Error: No CSV headers found in source file.")
                    return

                # 挿入するデータのリスト
                records_to_insert = []

                row_count = 0
                for row in reader:
                    row_count += 1
                    project_id = row.get('project_id')
                    budget_year = row.get('budget_year')
                    block_no = row.get('block_no')

                    if not (project_id and budget_year and block_no):
                        print(f"Skipping row {row_count} due to missing project_id, budget_year, or block_no: {row}")
                        continue

                    try:
                        project_id = int(project_id)
                        budget_year = int(budget_year)
                    except ValueError:
                        print(f"Skipping row {row_count} due to invalid project_id or budget_year: {row}")
                        continue

                    # block_id を project_spending_block テーブルからルックアップ
                    cur.execute(
                        "SELECT block_id FROM project_spending_block WHERE project_id = %s AND budget_year = %s AND block_no = %s",
                        (project_id, budget_year, block_no)
                    )
                    block_id_result = cur.fetchone()

                    if not block_id_result:
                        print(f"Skipping row {row_count}: No matching block_id found for project_id={project_id}, budget_year={budget_year}, block_no='{block_no}'")
                        continue
                    
                    block_id = block_id_result[0]

                    # その他のフィールドの準備
                    recipient_name = row.get('recipient_name')
                    corporate_number = row.get('corporate_number')
                    address = row.get('address')
                    corporation_type = row.get('corporation_type')
                    is_other_recipient = row.get('is_other_recipient') == 'True'
                    amount = row.get('amount')
                    contract_method = row.get('contract_method')
                    contract_method_detail = row.get('contract_method_detail')
                    bidders_count = row.get('bidders_count')
                    successful_bid_rate = row.get('successful_bid_rate')
                    single_bid_reason = row.get('single_bid_reason')
                    is_other_contract = row.get('is_other_contract') == 'True'
                    contract_summary = row.get('contract_summary')

                    # 数値型フィールドのNone変換
                    try:
                        amount = float(amount) if amount else None
                    except ValueError:
                        amount = None
                    try:
                        bidders_count = int(bidders_count) if bidders_count else None
                    except ValueError:
                        bidders_count = None
                    try:
                        successful_bid_rate = float(successful_bid_rate) if successful_bid_rate else None
                    except ValueError:
                        successful_bid_rate = None

                    records_to_insert.append((
                        block_id, project_id, budget_year, recipient_name, corporate_number,
                        address, corporation_type, is_other_recipient, amount,
                        contract_method, contract_method_detail, bidders_count,
                        successful_bid_rate, single_bid_reason, is_other_contract, contract_summary
                    ))
                
                print(f"Processed {row_count} rows from source CSV. Prepared {len(records_to_insert)} records for insertion.")

                # データを一括挿入
                if records_to_insert:
                    insert_query = """
                        INSERT INTO project_spending (
                            block_id, project_id, budget_year, recipient_name, corporate_number,
                            address, corporation_type, is_other_recipient, amount,
                            contract_method, contract_method_detail, bidders_count,
                            successful_bid_rate, single_bid_reason, is_other_contract, contract_summary
                        ) VALUES %s
                    """
                    extras.execute_values(
                        cur, insert_query, records_to_insert,
                        page_size=1000 # チャンクサイズを調整してパフォーマンスを向上
                    )
                    conn.commit()
                    print(f"Successfully inserted {len(records_to_insert)} records into project_spending.")
                else:
                    print("No records to insert into project_spending.")

        except FileNotFoundError:
            print(f"Error: Source CSV file not found at {source_csv_path}")
        except psycopg2.Error as e:
            print(f"Database error: {e}")
            if conn:
                conn.rollback()
        except Exception as e:
            print(f"An unexpected error occurred: {e}")
        finally:
            if cur:
                cur.close()
            if conn:
                conn.close()

    if __name__ == "__main__":
        source_file = 'converted_data/converted_project_spending.csv'
        import_project_spending_data(source_file)
    ```

2.  **Pythonライブラリのインストール**:
    `psycopg2-binary` がインストールされていない場合は、以下のコマンドでインストールします。

    ```bash
    pip install psycopg2-binary
    ```

3.  **データベース接続情報の更新**:
    `import_project_spending.py` スクリプト内の以下のデータベース接続情報を、あなたの環境に合わせて更新してください。

    ```python
    DB_NAME = "gyousei_db" # あなたのデータベース名に置き換えてください
    DB_USER = "postgres"   # あなたのユーザー名に置き換えてください
    DB_PASSWORD = "your_password" # あなたのパスワードに置き換えてください
    ```

4.  **スクリプトの実行**:
    以下のコマンドでスクリプトを実行します。

    ```bash
    python3 import_project_spending.py
    ```

5.  **インポートの確認**:
    データが正しくインポートされたかを確認します。

    ```sql
    SELECT COUNT(*) FROM project_spending;
    SELECT * FROM project_spending LIMIT 10;
    ```

## 5. トラブルシューティング

*   **`psycopg2.OperationalError: fe_sendauth: no password supplied`**:
    `DB_PASSWORD` が正しく設定されていないか、`pg_hba.conf` の設定が不適切です。パスワードを確認するか、`pg_hba.conf` を編集してローカル接続を許可してください。
*   **`psycopg2.errors.ForeignKeyViolation`**:
    `project_spending_block` テーブルに、`project_spending` が参照しようとしている `project_id`, `budget_year`, `block_no` の組み合わせが存在しない場合に発生します。`project_spending_block` のインポートが完了しているか、またはデータに不整合がないか確認してください。
*   **`FileNotFoundError`**:
    CSVファイルのパスが間違っています。スクリプト内の `source_file` パスを確認してください。
*   **`KeyError`**:
    CSVファイルのヘッダー名がスクリプトで期待されているものと一致していません。CSVファイルのヘッダーを確認し、スクリプト内の `row.get('ヘッダー名')` の部分を修正してください。
