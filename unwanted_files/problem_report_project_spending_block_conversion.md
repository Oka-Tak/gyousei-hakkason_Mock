# `project_spending_block` 変換プログラムに関する問題報告書

## 1. 問題の概要

`project_spending_block` テーブルにデータを挿入するためのCSVファイルを生成する `convert_project_spending_block.py` スクリプトが、本番データ (`source_data/5-1_RS_2024_支出先_支出情報.csv`) を使用した場合に空のCSVファイルを出力します。テストデータ (`source_data/source_test_data/5-1_mockdata.csv`) では正常に動作していました。

## 2. 考えられる原因

1.  **ファイルパスの誤り**: スクリプト内で指定されている本番データのファイルパスが間違っている、またはファイルが存在しない。
2.  **CSVヘッダーの不一致**: 本番データのCSVファイルのヘッダー行が、スクリプトが期待するヘッダー（例: `事業年度`, `予算事業ID`, `支出先ブロック番号` など）と正確に一致していない。`csv.DictReader` はヘッダーをキーとして使用するため、不一致があるとデータを読み取れません。
3.  **エンコーディングの問題**: 本番データのCSVファイルのエンコーディングが `utf-8` ではない可能性がある。
4.  **必須データの欠落**: 本番データにおいて、`budget_year`, `project_id`, `block_no` のいずれかの必須フィールドが多くの行で欠落しているため、スクリプトのスキップロジックによって全ての行が除外されている。
5.  **データ型の変換エラー**: `budget_year` や `project_id` を `int` に変換する際に、予期せぬ非数値データが含まれており、多くの行がスキップされている。

## 3. 調査計画と修正方針

1.  **詳細なログ出力の追加**: `convert_project_spending_block.py` スクリプトに、ファイル読み込みの成功/失敗、ヘッダーの読み込み状況、各行の処理状況、スキップされた理由などを詳細に出力するログを追加します。
2.  **本番データCSVの再確認**: `source_data/5-1_RS_2024_支出先_支出情報.csv` の内容を直接確認し、ヘッダーの正確性、エンコーディング、および必須データの存在を確認します。特に、`csv.DictReader` が正しくヘッダーを認識しているかを確認します。
3.  **スクリプトの修正**: ログ出力とCSV内容の確認結果に基づき、ヘッダーの調整、エンコーディングの変更、またはデータ処理ロジックの改善を行います。

## 4. 次のステップ

上記の調査計画に基づき、まずスクリプトに詳細なログ出力を追加し、本番データで実行して問題の根本原因を特定します。
